# vue3

一个面向校园/本地配送场景的前后端一体化示例项目。前端使用 Vue 3 + Vite，后端入口是 Python Flask 服务 run.py，整体围绕“用户下单、骑手接单、商家经营”三类角色展开。

## 项目概览

这个仓库里同时包含前端界面和后端服务脚本。前端通过 /api 访问后端，开发环境下由 Vite 代理转发到 http://127.0.0.1:5000。登录态会保存在浏览器 localStorage 中，登录后根据角色自动跳转到对应工作台。

系统支持以下核心能力：

- 普通用户浏览商家、查看菜单、选择收货地址、下单、确认收货、评价商家和骑手。
- 骑手查看待接订单、接单、标记送达、查看历史履历、查看个人评价统计。
- 商家维护店铺资料、上下架菜品、查看订单、查看评价、切换营业状态，并查看经营分析报表。
- 全局右下角提供“帮助”按钮，可直接向内置大模型提问，适合查操作、查流程、查接口、查开发问题。

## 角色与能力

### 普通用户

- 浏览所有商家列表。
- 根据商家名称或简介进行筛选。
- 查看商家详情、菜品、评价和营业状态。
- 在个人中心维护收货地址。
- 下单时选择默认地址或指定地址。
- 查看进行中订单和历史订单。
- 对已完成订单进行收货确认和双向评价。
- 查看订单联系方式，用于与商家或骑手沟通。

### 骑手

- 查看新订单大厅中的待接订单。
- 一键接单，接单后进入我的配送。
- 标记订单已送达。
- 查看历史履历。
- 查看和更新个人信息。
- 查看自己的评价统计和最近评价。
- 查看订单联系信息。

### 商家

- 查看自己的店铺资料。
- 修改店铺名称、简介、营业状态和店铺头像。
- 新增、编辑、删除菜品。
- 为菜品设置价格、描述、图片和上下架状态。
- 查看自己的订单列表，并按状态筛选。
- 查看商家评价列表。
- 查看经营统计、趋势图和菜品分析。
- 查看订单联系方式。

## 开发者指南

### 技术栈

- 前端：Vue 3、Vue Router、Axios、Vite
- 后端：Flask、SQLAlchemy、PyJWT、PyMySQL
- 运行模式：前端开发服务器 + 后端 API 服务 + MySQL 数据库

### 目录结构

```text
.
├── run.py                # 后端 Flask 入口
├── config.py             # 后端配置
├── models.py             # ORM 与数据库模型
├── schema.sql            # 数据库初始化脚本
├── src/
│   ├── api.js            # Axios 实例与拦截器
│   ├── main.js           # Vue 入口
│   ├── router/           # 路由配置与角色守卫
│   ├── utils/            # 时间与 toast 工具
│   └── views/            # 各角色页面
├── public/
├── pic/                  # 上传图片静态目录
└── vite.config.js        # 前端开发代理配置
```

### 环境要求

- Node.js 20.19+，或更高的 20/22 LTS 版本。
- Python 3.10+。
- MySQL 数据库。
- 建议安装 VS Code 的 Vue 扩展，便于语法提示和模板检查。

### 安装依赖

前端依赖：

```sh
npm install
```

后端依赖需要你在 Python 环境中安装项目所需包，常见包括 Flask、SQLAlchemy、PyJWT、PyMySQL、Werkzeug。

### 本地启动顺序

1. 准备数据库并导入 schema.sql。
2. 配置数据库连接环境变量，或者沿用 config.py 中的默认值。
3. 启动后端服务。
4. 启动前端开发服务器。
5. 打开浏览器访问前端地址，登录后按角色进入对应页面。

### 启动后端

后端默认监听 127.0.0.1:5000。可直接执行：

```sh
python run.py
```

可选环境变量：

| 变量名 | 说明 | 默认值 |
| --- | --- | --- |
| SECRET_KEY | JWT 签名密钥 | delivery-dev-secret |
| DB_USER | 数据库用户名 | root |
| DB_PASSWORD | 数据库密码 | 110120119 |
| DB_HOST | 数据库地址 | 127.0.0.1 |
| DB_PORT | 数据库端口 | 3306 |
| DB_NAME | 数据库名称 | keshe |
| DATABASE_URL | 完整数据库连接串 | 由上面字段拼接 |
| IMG_PATH | 图片存储路径 | C:\Users\Tsing\Desktop\系统\server\imgs |
| PORT | 后端监听端口 | 5000 |
| FLASK_DEBUG | 是否开启调试模式 | 0 |

### 启动前端

```sh
npm run dev
```

Vite 会把 /api 请求代理到后端 http://127.0.0.1:5000，因此前端代码里统一使用 /api 前缀即可。

### 打包生产版本

```sh
npm run build
```

### 预览生产构建

```sh
npm run preview
```

### 开发者需要了解的实现细节

- 登录成功后，前端会把 user、token 和可选的 credentials 写入 localStorage。
- 路由守卫会根据 localStorage.user.role 自动限制页面访问，未登录时会跳转到登录页。
- 请求拦截器会优先附带 Authorization: Bearer <token>，旧逻辑下也保留了账号密码回填的兼容行为。
- 头像、菜品图等图片由后端保存后返回路径，前端会兼容 /pic/... 和 /api/pic/... 两种格式。
- 后端返回 401 时会自动清理登录态并跳回登录页，避免脏会话。

## 用户指南

### 1. 注册与登录

打开系统后会先进入登录页，页面支持两种模式：

- 登录：输入用户名或手机号 + 密码。
- 注册：输入用户名、手机号、密码，并选择角色。

注册时不同角色需要补充的信息略有不同：

- 普通用户可以选填收货地址。
- 商家必须填写店铺名称。
- 骑手只需填写基础账号信息。

登录页提供“记住我”选项，开启后会把账号密码保存在本机浏览器中，方便下次自动填充。该信息只保存在本地，不会同步到其他设备。

### 2. 普通用户使用说明

#### 浏览商家

登录后会自动进入用户工作台。首页展示商家卡片，每个商家会显示：

- 店铺名称。
- 简介。
- 订单量。
- 平均评分。
- 营业状态。

你可以通过搜索框按商家名称或简介筛选结果。列表中营业中的商家会排在前面，休息中的商家会保留展示，但不能直接下单。

#### 查看商家详情并下单

点击商家卡片后进入详情页。这里可以看到：

- 商家基础信息和营业状态。
- 菜品列表。
- 菜品图片、描述和价格。
- 历史评价。

下单步骤如下：

1. 点击菜品右侧的加号把菜品加入购物车。
2. 在购物车中调整数量。
3. 从已有收货地址中选择一个地址。
4. 点击去结算完成下单。

如果商家处于休息状态，页面会允许浏览菜单，但不会允许提交订单。

#### 管理收货地址

在用户中心可以维护多个地址。地址信息包括：

- 地址标签，例如宿舍、家、实验室。
- 收件人姓名。
- 收件人手机号。
- 详细地址。
- 是否默认地址。

提交订单时会优先使用默认地址；如果没有默认地址，则会使用你当前选择的地址。

#### 处理订单

用户中心的订单页会分成两部分：

- 进行中订单：等待骑手接单、配送中、已送达但未确认等状态。
- 历史订单：已完成或已评价的订单。

你可以在这里：

- 刷新订单状态。
- 确认收货。
- 对已完成订单进行评价。
- 查看订单相关联系方式。

#### 评价订单

当骑手完成送达并且你确认收货后，订单会进入可评价状态。评价包括两个维度：

- 商家评分与评论。
- 骑手评分与评论。

提交后订单会进入已评价状态，避免重复评价。

### 3. 骑手使用说明

#### 查看可接订单

骑手登录后会进入骑手工作台，默认显示新订单大厅。这里的订单是状态为待接单的实时订单。

每个订单会显示：

- 订单号。
- 下单时间。
- 订单金额。
- 起点商家。
- 送达地址。

点击立即接单后，订单会从大厅中消失，并进入你的我的配送列表。

#### 配送流程

骑手的配送流程是：

1. 接单。
2. 送餐。
3. 标记送达。
4. 用户确认收货后，订单进入历史记录。

系统会限制订单状态流转，防止重复接单或越级修改状态。

#### 个人中心与评价

骑手可以在个人中心：

- 修改用户名。
- 修改手机号。
- 修改密码。
- 查看自己的平均评分。
- 查看最近的评价记录。

如果用户或商家在订单里留下了评价，骑手可以在这里看到反馈情况。

### 4. 商家使用说明

#### 店铺资料

商家工作台会展示自己的店铺基础信息，并允许编辑：

- 店铺名称。
- 店铺简介。
- 店铺头像。
- 营业状态。

营业状态决定用户是否可以继续下单。休息中的商家仍然能被浏览，但不能被新订单下单。

#### 菜品管理

商家可以在菜品管理页进行：

- 新增菜品。
- 编辑菜品名称、价格、描述、图片和上下架状态。
- 删除未产生订单记录的菜品。

建议的使用方式是：

- 已经有订单记录的菜品优先下架，不要删除。
- 保持价格和库存状态及时更新，避免用户下单失败。

#### 订单管理

商家可以查看自己的所有订单，并按状态筛选。订单列表里会显示：

- 订单编号。
- 创建时间。
- 顾客名称。
- 当前状态。
- 菜品明细。
- 骑手信息。
- 订单总价。

商家还可以查看订单联系方式，便于处理异常配送、顾客沟通或骑手协同。

#### 经营分析

商家工作台的经营分析会展示：

- 不同周期的订单量和营业额统计。
- 趋势图。
- 统计表格。
- 菜品销量与营收分析。

这些分析主要基于已完成和已评价订单，用于查看真实经营结果。

## 常见问题

### 为什么登录后又回到登录页？

通常是 token 过期、localStorage 中的数据不完整，或者后端返回了 401。可以先清理浏览器里的 user、token、credentials 后重新登录。

### 前端请求为什么报 404？

先确认后端已经启动，其次确认 Vite 的代理生效。开发模式下前端发起的请求应该走 /api 前缀，不要手动改成后端完整地址。

### 图片为什么显示不出来？

后端会返回 /pic/... 或 /api/pic/... 两种路径，前端已做兼容。如果仍然无法显示，请检查：

- 图片文件是否真的上传成功。
- pic/ 目录是否可读。
- 后端图片保存路径是否正确。

### 数据库连接失败怎么办？

检查 DB_HOST、DB_PORT、DB_USER、DB_PASSWORD、DB_NAME 是否正确，以及 MySQL 服务是否已经启动。

## 推荐开发流程

1. 先运行数据库初始化脚本，确认表结构存在。
2. 启动后端，确保 / 和 /stats 能正常响应。
3. 启动前端，检查登录页是否能正常提交。
4. 按角色分别验证用户、骑手、商家三条主流程。
5. 变更接口后，优先核对 src/api.js 和 src/router/index.js 的联动关系。

## 构建与部署提示

- 开发阶段优先使用 npm run dev。
- 生产环境先执行 npm run build，再由静态服务器托管前端资源。
- 后端建议使用独立进程运行，并通过反向代理把 /api 转发到 Flask 服务。
